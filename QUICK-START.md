## 3-Minute Deployment to GitHub Pages

### Step 1: Create GitHub Repository (1 minute)
1. Go to https://github.com/new
2. Repository name: `leading-website` (or your choice)
3. Select: **Public**
4. Click: **Create repository**

### Step 2: Upload Files (1 minute)
1. On the new repository page, click: **uploading an existing file**
2. Drag ALL 7 files from your `leading-website` folder into the browser:
   - index.html
   - new-leader.html
   - resources.html
   - programs.html
   - continuing-education.html
   - logo.png
   - README.md
3. Click: **Commit changes**

### Step 3: Enable GitHub Pages (1 minute)
1. Click: **Settings** tab (top of repository)
2. Click: **Pages** (left sidebar)
3. Under "Source": Select **main** branch
4. Click: **Save**
5. Wait 1-2 minutes for deployment

### Your Website URL
```
https://[YOUR-GITHUB-USERNAME].github.io/leading-website/
```

Example: If your username is `jpaylocity`, your site will be:
```
https://jpaylocity.github.io/leading-website/
```

## Testing Locally (Optional)

To test on your computer before deploying:

1. Download all files to a folder

## Need Help?

Common issues:
- **404 Error**: Wait 2-3 minutes after enabling Pages
- **Images Not Loading**: Make sure `logo.png` was uploaded
- **Links Broken**: Check that all filenames are lowercase with hyphens
